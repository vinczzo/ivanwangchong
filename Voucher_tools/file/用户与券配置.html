指定用户（必填）

1. 【导入用户数据】和【下载模板】按钮

2. 导入进度条，以及导入提示文案：“正在导入，请勿退出...“

3. 多次导入时，采用覆盖的导入方式，即每次导入，均只保存最新导入的数据，删除之前导入的记录

4. 导入数据统计表：

（1）导入条数：导入表格中，所有数据的条数统计

（2）用户数：导入的表格中，独立的用户数量（即排除重复记录后的数量）

- 交互说明：

（1）点击【导入用户数据】后，调用系统文件选择弹窗，可选excel格式文件；选择文件并导入时校验表格中数据格式是否和模板一致，若不一致，toast提示”文件有误，请重新导入“；若文件选择无误，则导入数据，并显示进度条，导入完成后进度条及提示文案隐藏，显示导入结果的数据统计表

（2）点击【下载模板】后，盗用系统弹窗，选择文件下载位置并下载文件

点击后保存当前页面配置的信息，并返回“基本信息”配置页面

发布

校验

1. 必填项是否已填写

2. 已填写项是否符合输入规则

3. 已选优惠券批次的发放数量大于等于导入记录条数

4. 已选批次的优惠券，领取方式为“自动领取”

- 符合，弹窗进行二次确认，确定后将该活动状态变革为正在发放；根据选择的优惠券和导入的用户信息，开始向用户发放优惠券；返回活动列表并定位至该活动所在位置

- 不符合，toast提示“您选择的优惠券有误，请确认后再发布”

保存

校验

1. 必填项是否已填写

2. 已填写项是否符合输入规则

3. 已选优惠券批次的发放数量大于等于导入记录条数

4. 已选批次的优惠券，领取方式为“自动领取”

- 符合则保存所有配置信息；返回活动列表定位至该活动

- 不符合，则toast提示“您选择的优惠券有误，请确认后再保存”

取消

——

点击后弹窗二次确认，确定后返回活动列表，不保存该活动任何信息

4

定向发券规则：

1. 按照导入的用户列表，逐条发放优惠券，每条记录中的用户ID发放一张

2. ID重复，正常向其发放优惠券；若超出优惠券发放的ID数量限制，则记录为发放失败，失败原因“ID领取数量限制”

3. 若用户ID不正确，记录为发放失败，失败原因“用户ID错误”

4. 若自动发放过程中，发生问题导致发送中断，记录未发送的记录为发送失败，失败原因“系统错误”
